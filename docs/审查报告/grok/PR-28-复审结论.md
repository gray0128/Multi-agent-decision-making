## 工作区复审：新增 / 残留 Findings

### R1 · [中] `ARCHIVE_NOT_READY` 标 failed 后释放 pending 槽，仍在跑的子进程可能被二次启动抢道

- **位置**：`src/server/launch-coordinator.ts` `withArchiveStatus`（~188–194）+ `pendingDeliberationId`（仅 `reserved|spawned`）
- **路径**：spawn 成功但 >10s 未建档 → 记录 `failed` + `ARCHIVE_NOT_READY` → `canLaunch` 变 true → 新 `requestId` 可再 launch；首进程随后拿锁或双进程竞态
- **影响**：同 requestId 映射可自愈（已测），**跨 requestId** 窗口仍可能冲突或浪费资源
- **建议**：`ARCHIVE_NOT_READY` 在尚无档案时仍视为 pending（或独立 `lost` 态且占用活动语义）；或记录 child pid 探测存活后再放行

### R2 · [低–中] 控制台方案检查点无「暂停」按钮

- **位置**：`src/web/index.ts` `planEditor` 仅校验/重新生成/确认/取消
- **影响**：终端 Ctrl-C / 信号路径已正确 pause；网页用户只能 cancel（终态）。领域「暂停 ≠ 取消」在 UI 仍不完整
- **建议**：增加 pause 动作按钮（后端 actions 已含 pause）

### R3 · [低] `startLaunch` 仍无限轮询

- **位置**：`src/web/index.ts` `while(true)` + 500ms
- **影响**：长期 `spawned` 时按钮锁定、无取消；可依赖 R1 后的 failed 文案退出，体验仍弱
- **建议**：超时上限 + 取消/离开保护

### R4 · [低] 失败记录幂等重试仍统一 HTTP 200

- **位置**：`observer.ts` 已存在 launch 记录一律 200
- **影响**：客户端须读 `status`；文档/契约可写清（非正确性阻断）

### R5 · [信息] 本地未提交修复尚未进入 PR tip

- **影响**：CI / 协作者 / GitHub review 仍只见 `5ec1ac2`；复审「工作区通过」≠「PR 已修复」
