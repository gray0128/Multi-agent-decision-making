## 跨智能体共识：必须先修（Blocking / High）

### B1 · plan_confirmation 不接受 `pause`，abort 挂死信箱

- **来源**：启动协调审查 F1/F5；与 CONTEXT「暂停 ≠ 取消」冲突
- **位置**：`src/cli/index.ts`（actions 仅 confirm|replace|regroup|cancel）；`src/server/mailbox.ts`（abort 写 `pause` + 动作白名单拒绝）
- **失败路径**：guided / `--web-plan` → Ctrl-C / abort → 写入不可消费的 `pause` → 轮询永不结束；残留响应可污染 resume
- **最小修复**：`actions` 含 `"pause"`；`confirmPlan` 收到 pause 时抛 `PAUSED`；abort 动作不在 actions 时不得独占信箱

### B2 · 启动记录 10s 无档案即粘性 `failed`，永不自愈

- **来源**：启动协调审查 F2
- **位置**：`src/server/launch-coordinator.ts` `withArchiveStatus`（failed 短路 + 超时标 failed）
- **失败路径**：spawn 成功但建档慢 → 轮询/冲突扫描触发 read → 记 failed → 子进程随后正常规划；同 requestId 永久返回失败
- **最小修复**：超时与硬失败分离；`failed`（无档案类）仍探测档案并可升级；阈值放宽或仅告警

## 安全面结论（摘要）

- **无阻断级**：无已证实 RCE、鉴权绕过、信箱重放接管、控制服务写正式档案
- **成立项**：Bearer 覆盖新 API；spawn 用 `execPath` + argv 无 shell；公开视图无 executable；spawn 失败路径脱敏；信箱一次性
- **Warning**：W1 topic argv；W2 workspace 预检；W3 顶层 500 原样 `error.message`
- **Note**：requestId 不绑定 body 指纹；validationError 统一 redact 可选；档案 detail 仍含 registry executable（既有观察面）
