# Claude Code Agent 信息收集

本文件记录 Claude Code（本会话运行环境）在 Agent / Workflow 调用中可用的模型与思考（推理强度）等级。

## 1. 可用模型

在 `Agent` 工具的 `model` 参数、`Workflow` 脚本的 `agent({model})` 等位置可指定以下模型枚举值：

| 枚举值 | 定位 | 适用场景 |
|--------|------|----------|
| `opus` | 能力最强 | 复杂推理、困难任务、最高质量要求 |
| `sonnet` | 均衡型 | 速度与能力兼顾，常见默认选择 |
| `haiku` | 轻量最快 | 简单机械任务、低成本步骤 |
| `fable` | 可选模型 | 环境列出的备选模型 |

说明：不显式指定 `model` 时，子任务/子 agent 默认继承会话当前解析出的模型（`fork` 类型强制继承父模型）。

## 2. 思考等级（reasoning effort）

在 `Workflow` 的 `agent({effort})`、`ReportFindings.level`、`ScheduleWakeup` 等位置可用的推理强度枚举：

| 等级 | 适用场景 |
|------|----------|
| `low` | 廉价、机械的步骤 |
| `medium` | 常规任务 |
| `high` | 较难的分析 / 校验 |
| `xhigh` | 高难度、需要深度推理 |
| `max` | 最高强度，用于最难的验证 / 评审阶段 |

说明：不指定 `effort` 时继承会话当前的 effort 设置。

## 3. 实际运行时环境变量

本环境实际通过环境变量将上述模型枚举映射到具体后端模型（MiniMax Anthropic 兼容接口）：

| 环境变量 | 取值 |
|----------|------|
| `ANTHROPIC_BASE_URL` | `https://api.minimaxi.com/anthropic` |
| `ANTHROPIC_MODEL` | `MiniMax-M3` |
| `ANTHROPIC_REASONING_MODEL` | `MiniMax-M3` |
| `ANTHROPIC_DEFAULT_HAIKU_MODEL` | `MiniMax-M3` |
| `ANTHROPIC_DEFAULT_SONNET_MODEL` | `MiniMax-M3[1M]` |
| `ANTHROPIC_DEFAULT_OPUS_MODEL` | `MiniMax-M3[1M]` |
| `ANTHROPIC_DEFAULT_FABLE_MODEL` | `MiniMax-M3[1M]` |
| `CLAUDE_CODE_EFFORT_LEVEL` | `max` |
| `CLAUDE_CODE_AUTO_COMPACT_WINDOW` | `1000000` |
| `API_TIMEOUT_MS` | `3000000` |
| `ENABLE_TOOL_SEARCH` | `true` |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` | `1` |

要点：
- 四个模型档位（haiku/sonnet/opus/fable）实际都指向 `MiniMax-M3`，其中 sonnet/opus/fable 使用 `[1M]`（百万级上下文）变体。
- 默认会话 effort 为 `max`。

## 4. Models.dev 核验与口径修正

| 路由 | Context | Max output | 口径 |
|---|---:|---:|---|
| `MiniMax-M3` 规范模型 | 512,000 | 128,000 | Models.dev 规范页 |
| MiniMax 官方 provider 的 `MiniMax-M3` | 1,000,000 | 128,000 | Models.dev provider 行 |
| 本环境 `MiniMax-M3[1M]` | 1,000,000 | 128,000 | 与 MiniMax provider 行相符 |

因此，“MiniMax-M3 一律是 1M 上下文”不准确；1M 是当前 provider / 路由能力，规范模型页的默认 Context 为 512,000。实际运行还应服从 Anthropic 兼容网关与 Claude Code 自动压缩策略。详见 [Models.dev 核验记录](../models-dev核验.md)。
