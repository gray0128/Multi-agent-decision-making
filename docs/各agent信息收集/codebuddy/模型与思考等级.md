# CodeBuddy Code Agent 信息收集

本文件记录 CodeBuddy Code（本会话运行环境）在 Agent / 子代理调用中可用的模型与思考（推理强度）等级，以及非交互 CLI（headless / `--print`）模式下的重点参数。

- 采集命令：`/model list`、`codebuddy --help`
- 文档参考：内置文档 `docs/cn/cli/models.md`、`settings.md`、`cli-reference.md`
- 模型配置文件：`~/.codebuddy/models.json`（用户级）、`<workspace>/.codebuddy/models.json`（项目级）

## 1. 可用模型

> 本文档仅保留下述两个作为本机接入 Multi-agent 决策框架时使用的模型。

| 模型 ID | 显示名 | 说明 |
|---------|--------|------|
| `hy3` | Hy3 | 当前默认模型；综合能力强，适合复杂推理与多步任务 |
| `kimi-k3-2` | Kimi-K3 | 长上下文 / 长文处理场景，适合大文档分析与摘要 |

切换方式：

- 交互模式：`/model hy3` 或 `/model kimi-k3-2`；`/model list` 列出全部可见模型
- 非交互 CLI：`codebuddy --model hy3 "..."`
- 持久默认：`settings.json` 的 `"model": "hy3"`，或 `codebuddy config set model hy3`

说明：CodeBuddy 完整模型清单由账号权限与 `models.json` 决定（`/model list` 实时为准）。除上述两个外，本机还可见 GLM / MiniMax / DeepSeek 等多个内置模型，但本框架仅选用 Hy3 与 Kimi-K3。

## 2. 思考等级（reasoning effort）

控制模型推理深度，共以下控制点：

| 参数 | 位置 | 可选值 | 说明 |
|------|------|--------|------|
| `reasoningEffort` | `settings.json` 或 `/config` 面板 | `low` / `medium` / `high` / `xhigh` | 推理深度。留空用产品默认；`auto` 等效清除 |
| `alwaysThinkingEnabled` | `settings.json` | `true` / `false` | 始终启用思考模式 |
| `MAX_THINKING_TOKENS` | 环境变量（或 `settings.json` 的 `env`） | 数字字符串（如 `"10000"`） | 思考 token 预算上限，扩展推理长度 |
| `supportsReasoning` | 模型定义（`models.json`） | `true` / `false` | 声明该模型是否支持推理模式 |

非交互模式下设置思考等级的方式：

```bash
# 通过 --settings 内联 JSON 设置推理深度
codebuddy -p "深度分析这个模块" --model hy3 --settings '{"reasoningEffort":"high"}'

# 通过环境变量设置思考 token 预算
MAX_THINKING_TOKENS=10000 codebuddy -p "..." --model hy3
```

## 3. 非交互 CLI 模式重点参数

非交互模式以 `-p / --print`（执行后退出）为主，常用于脚本与自动化。重点参数如下：

### 3.1. 模型与思考相关

| 参数 | 说明 | 示例 |
|------|------|------|
| `--model <model>` | 指定本次会话模型（须提供模型 ID） | `codebuddy -p "..." --model hy3` |
| `--fallback-model <model>` | 默认模型过载时自动回退；**仅 `--print` 模式生效** | `codebuddy -p "..." --fallback-model kimi-k3-2` |
| `--text-to-image-model <model>` | 文生图功能使用的模型 ID | `--text-to-image-model <id>` |
| `--image-to-image-model <model>` | 图生图功能使用的模型 ID | `--image-to-image-model <id>` |
| `--settings <json>` | 内联 JSON 设置（可含 `reasoningEffort` 等） | `--settings '{"reasoningEffort":"high"}'` |

### 3.2. 输出与执行控制

| 参数 | 说明 | 示例 |
|------|------|------|
| `-p, --print` | 打印响应后退出，不进入交互 | `codebuddy -p "查询"` |
| `--output-format <fmt>` | 输出格式：`text` / `json` / `stream-json` | `--output-format json` |
| `--input-format <fmt>` | 输入格式：`text` / `stream-json` | `--input-format stream-json` |
| `--include-partial-messages` | 输出含部分流式事件（需 `--print` + `--output-format stream-json`） | `--output-format stream-json --include-partial-messages` |
| `--json-schema <schema>` | 用 JSON Schema 校验结构化输出 | `--json-schema '{...}'` |
| `--max-turns <n>` | 限制非交互模式下的代理轮次 | `--max-turns 3` |
| `--verbose` / `--debug` | 详细 / 调试日志 | `codebuddy --debug` |

### 3.3. 权限（非交互模式必需）

非交互执行涉及文件读写、命令执行、网络请求时，必须明确权限策略，否则需人工确认的操作会被阻止。

| 参数 | 说明 | 示例 |
|------|------|------|
| `-y, --dangerously-skip-permissions` | 跳过权限提示（谨慎使用） | `codebuddy -y -p "..."` |
| `--permission-mode <mode>` | 权限模式：`default` / `acceptEdits` / `auto` / `dontAsk` / `plan` / `bypassPermissions` | `--permission-mode auto` |
| `--allowedTools <tools...>` | 免确认允许的工具白名单 | `"Read" "Edit" "Bash(git:*)"` |
| `--disallowedTools <tools...>` | 禁止使用的工具黑名单 | `"Bash(rm:*)"` |
| `--permission-prompt-tool <tool>` | 非交互下处理权限提示的 MCP 工具 | `--permission-prompt-tool mcp_auth_tool` |

### 3.4. 上下文与系统提示

| 参数 | 说明 | 示例 |
|------|------|------|
| `--system-prompt <text>` | 替换整个系统提示词 | `--system-prompt "你是 Python 专家"` |
| `--append-system-prompt <text>` | 在默认提示词末尾追加 | `--append-system-prompt "始终用 TypeScript"` |
| `--agent <name>` | 指定会话使用的 agent（覆盖默认） | `--agent my-reviewer` |
| `--add-dir <dirs...>` | 添加额外可访问工作目录 | `--add-dir ../apps` |
| `--resume <id>` / `-c` | 恢复指定会话 / 继续最近会话 | `codebuddy -c -p "..."` |

常用速查：

```bash
# 非交互：指定模型 + 推理深度 + 自动放行权限
codebuddy -p "实现并测试登录功能" --model hy3 \
  --settings '{"reasoningEffort":"high"}' --permission-mode auto

# 非交互：结构化 JSON 输出（适合脚本消费）
codebuddy -p "提取关键信息" --model hy3 --output-format json \
  --json-schema '{"type":"object","properties":{"name":{"type":"string"}},"required":["name"]}'

# 非交互：长文分析走 Kimi-K3，超限自动回退
codebuddy -p "总结这份长文档" --model kimi-k3-2 --fallback-model hy3
```

## 4. Models.dev 映射核验

| CodeBuddy ID | Models.dev 候选 | 候选 Context / Output | 结论 |
|---|---|---:|---|
| `hy3` | `tencent/hy3-preview` | 256,000 / 64,000 | 名称接近，但缺少 CodeBuddy 别名到规范 ID 的明确证据 |
| `kimi-k3-2` | `moonshotai/kimi-k3` | 1,048,576 / 131,072 | 名称接近，但 `-2` 后缀含义未证实 |

在取得 `/model list` 的完整模型元数据或 `models.json` 映射前，不把候选窗口写成 CodeBuddy 的已验证限制。详见 [Models.dev 核验记录](../models-dev核验.md)。
