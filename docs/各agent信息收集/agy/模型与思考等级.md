# Antigravity (AGY) Agent 信息收集

本文件记录 Google Antigravity (AGY) Agent 运行环境中可用的模型、思考等级（Reasoning Effort / Thinking Depth）及相关系统参数。

## 1. 可用模型 (Model Selection)

在 Antigravity (AGY) 界面设置、命令行配置或子 Agent 调度 (`invoke_subagent`) 中，可选择以下模型类型：

| 模型标识 / 名称 | 定位 | 适用场景 |
| :--- | :--- | :--- |
| `Gemini 3.5 Pro` (`pro`) | 旗舰推理模型 | 复杂架构设计、大规模重构、高难度算法与复杂调试 |
| `Gemini 3.5 Flash` (`flash`) | 速度与性能均衡 | 日常功能开发、代码修改、文档编写与单元测试 |
| `Gemini 3.5 Flash-Lite` (`flash_lite`) | 极速轻量模型 | 快速日志扫描、简单文件检索与低成本工具调用 |
| `Auto / Inherit` (`inherit`) | 继承/自动调度 | 默认模式，自动继承主 Agent 的模型或由系统动态匹配 |

---

## 2. 思考等级 (Thinking Level / Reasoning Effort)

思考等级控制模型在生成输出或调用工具前进行链式思考（Chain-of-Thought）的深度：

| 思考等级 | 响应延迟 | 适用场景与行为特点 |
| :--- | :--- | :--- |
| `Off / Minimal` | 极低 | 语法问答、单文件微调、快速格式转换，不消耗额外推理 Token |
| `Low` | 低 | 快速代码修改、简单函数生成，进行基础逻辑检查 |
| `Medium` | 平衡 | 一般功能模块开发、跨文件修改，兼顾速度与质量 |
| `High` | 高 | 复杂 Bug 调试、架构重构设计、安全性与边界条件推演 |

---

## 3. 子 Agent 调度模型参数 (Subagent Tier)

在调用子 Agent (`invoke_subagent`) 时可传递 `Model` 参数控制子任务的资源消耗：

- `inherit`（默认）：继承主 Agent 相同的模型与思考等级配置。
- `pro`：为特定高难度子任务独立调度 Pro 级大模型。
- `flash`：为一般检索与文件修改任务调度 Flash 级模型。
- `flash_lite`：为只读查询、快速搜索调度轻量 Flash-Lite 模型。

---

## 4. 单次调用重点参数 (Single Invocation / Execution Parameters)

在 SDK (`google.antigravity.Agent` / `LocalAgentConfig`)、子任务派生 (`invoke_subagent` / `define_subagent`) 及底层工具执行控制中，单次调用的核心重点参数如下：

### 4.1 Agent 实例与能力配置参数 (`LocalAgentConfig` / `define_subagent`)
| 参数名 | 类型 / 取值范围 | 说明与控制作用 |
| :--- | :--- | :--- |
| `system_instructions` / `system_prompt` | `string` | 单次 Agent 实例的系统级 Prompt，定义角色定位、代码规范与行为约束边界。 |
| `capabilities` | `CapabilitiesConfig` | 单次调用的工具能力权限集：<br>• `enable_write_tools` (`bool`)：控制写文件/命令行运行权限。<br>• `enable_mcp_tools` (`bool`)：控制 MCP (Model Context Protocol) 服务工具加载。<br>• `enable_subagent_tools` (`bool`)：控制是否允许进一步派生/调用子 Agent。 |
| `workspace` | `enum`<br>(`inherit` \| `branch` \| `share`) | 单次调用的工作区隔离策略：<br>• `inherit`：共享当前环境目录。<br>• `branch`：克隆全新隔离分支。<br>• `share`：以共享存储方式创建独立分支工作区。 |

### 4.2 单次对话与派生任务参数 (`agent.chat()` / `invoke_subagent`)
| 参数名 | 类型 / 取值范围 | 说明与控制作用 |
| :--- | :--- | :--- |
| `prompt` / `Prompt` | `string` | 传递给 Agent 的单次主任务指令或具体交互需求。 |
| `model` / `Model` | `enum`<br>(`inherit` \| `pro` \| `flash` \| `flash_lite`) | 指定该单次任务运行的模型档位（未显式指定时默认 `inherit` 继承父级）。 |
| `role` / `Role` | `string` | 单次派生任务中的 Agent 岗位/角色标识（如 `Codebase Researcher`）。 |
| `timeout_ms` / `timeout` | `integer` (毫秒) | 单次调用的超时时间门限，防止长时间挂起或死循环。 |

### 4.3 单次工具执行与控制参数 (`run_command` / `schedule`)
| 参数名 | 类型 / 取值范围 | 说明与控制作用 |
| :--- | :--- | :--- |
| `WaitMsBeforeAsync` | `integer`<br>(`0` - `10000` ms) | 终端命令单次执行时由同步等待转为后台 Task 的延迟门限。若在此毫秒数内未完成，自动转后台异步执行。 |
| `TimerCondition` | `enum`<br>(`never` \| `any` \| `<sender-id>`) | 单次唤醒定时器的提前终止触发条件（收到任一消息或指定 Sender 消息时提前取消）。 |

## 5. 配置文件与路径

- **配置路径**：`~/.gemini/antigravity-cli/settings.json`
- **应用数据目录**：`~/.gemini/antigravity-cli`
- **技能/插件扩展**：`~/.gemini/skills/` 与 `~/.gemini/config/plugins/`
- **控制台 CLI 工具**：`agy`
