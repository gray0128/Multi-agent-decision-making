## 2. 思考 / 推理等级参数

Grok 存在 **两套** 相关参数，合法取值不完全相同，用途也不同。

### 2.1 `--reasoning-effort`（推理思考强度）

面向支持 reasoning 的模型（当前 `grok-4.5` 的 `supports_reasoning_effort = true`）。

**合法取值**（由 CLI 非法值错误信息确认）：

| 等级 | 说明（经验理解） |
|------|------------------|
| `none` | 关闭 / 不做推理链 |
| `minimal` | 最低思考 |
| `low` | 低 |
| `medium` | 中 |
| `high` | 高（当前本机默认） |
| `xhigh` | 很高 |

**注意：** `max` **不是** `--reasoning-effort` 的合法值。

**用法：**

```bash
grok -p "分析这个 bug" -m grok-4.5 --reasoning-effort high
```

**持久默认：**

```toml
[models]
default_reasoning_effort = "high"  # none | minimal | low | medium | high | xhigh
```

TUI 界面存在 reasoning effort 选择器（文案：*Pick reasoning effort*）。

### 2.2 `--effort`（Agent 努力等级，无头模式）

控制 agent 整体投入程度。文档标注为 **headless only**；在交互 TUI 中使用会打印告警并被忽略。

**合法取值**（由 CLI help / 非法值错误信息确认）：

| 等级 |
|------|
| `low` |
| `medium` |
| `high` |
| `xhigh` |
| `max` |

**注意：** 不含 `none`、`minimal`；**含** `max`。

**用法：**

```bash
grok -p "实现这个功能" --effort max
```

### 2.3 两套参数对比

| 项目 | `--reasoning-effort` | `--effort` |
|------|----------------------|------------|
| 语义 | 推理思考链强度 | Agent 任务投入度 |
| 合法值 | `none`, `minimal`, `low`, `medium`, `high`, `xhigh` | `low`, `medium`, `high`, `xhigh`, `max` |
| 有 `none` / `minimal` | 是 | 否 |
| 有 `max` | 否 | 是 |
| 模式限制 | 通用 CLI 参数 | 文档标明 headless only |
| 配置项 | `[models].default_reasoning_effort` | 无对应全局 config 键（以 CLI 传入为主） |
| 本机默认 | `high` | 未在 config 中固定 |

### 2.4 其他可覆盖入口

| 位置 | 字段 | 作用 |
|------|------|------|
| 模型目录 / 缓存 | `reasoning_effort` | 单模型默认思考强度 |
| 模型目录 / 缓存 | `supports_reasoning_effort` | 是否支持调整思考强度 |
| Skill frontmatter | `effort` | 该 skill 的 reasoning-effort 覆盖（见 `08-skills.md`） |
| Persona / Subagent | `reasoning_effort` | 使用该 persona 时的推理强度（见 `16-subagents.md`） |

Persona 生效时，模型与 reasoning effort 的解析优先级以官方 subagents 文档为准（persona 覆盖高于默认，再落到会话模型）。

## 4. 常用命令速查

```bash
# 列出可用模型
grok models

# 指定模型 + 推理强度（无头）
grok -p "你的任务" -m grok-4.5 --reasoning-effort high

# 指定 agent 努力度（无头）
grok -p "你的任务" --effort max

# 同时指定模型与两类等级（无头）
grok -p "你的任务" -m grok-4.5 --reasoning-effort xhigh --effort high
```

TUI 内：

```
/model grok-4.5
/m grok-4.5
```

## 5. Models.dev 核验

`grok-4.5` 与 Models.dev 的 `xai/grok-4.5` 精确匹配。Models.dev 规范页及其 xAI provider 行给出 Context 500,000、Output limit 500,000，并标记支持 reasoning 与 tool call；本机缓存也记录相同值。xAI 公开模型页独立确认了 500,000 Context，但没有单列最大输出，因此 Output 500,000 暂按中等置信度记录。不同第三方 provider 的输出上限可能更小，跨 provider 使用时需按具体接入值配置。详见 [Models.dev 核验记录](../models-dev核验.md)。
