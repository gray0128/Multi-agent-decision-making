# 采用结构化审议

结构化审议中，所有参与者先独立陈述，再基于完整记录质疑与补充，随后提交修订意见；若满足按需条件则追加一次有严格范围的争议收敛轮，然后由报告 Agent 草拟报告、其他参与者审阅，最后修订一次并形成共同成果。参与者阶段均在阶段内逻辑并行执行，并以屏障等待全部调用成功、失败或超时后再进入下一阶段；同一阶段的参与者不会看到彼此尚未完成的输出。应用同时提供自动模式和引导模式，`mad deliberate` 默认使用引导模式，只有显式 `--auto` 才跳过正常阶段检查点；方案自动确认仍需独立的 `--auto-confirm-plan`。引导模式在独立陈述、质疑补充、争议判定及报告草稿后设置检查点，终端和观察页提交的第一份有效响应获胜；既无交互终端也无在线观察服务时立即报错，模式写入档案且恢复时不得切换。相比自由讨论，该方案具有明确的参与范围、成本上限、终止条件和恢复边界；代价是流程更长，且争议判定需要结构化信号和额外整理。
