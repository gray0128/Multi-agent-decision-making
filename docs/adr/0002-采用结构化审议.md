# 采用结构化审议

MVP 采用固定阶段的结构化审议：所有参与者先独立陈述，再基于完整发言记录进行质疑与补充，随后提交修订意见；指定的报告 Agent 生成报告草稿，其他参与者完成一次报告审阅，最后由报告 Agent 修订一次并形成共同成果。独立陈述、质疑补充、修订意见和报告审阅均在阶段内并行执行，并以屏障等待本阶段全部调用成功、失败或超时后再进入下一阶段；同一阶段的参与者不会看到彼此尚未完成的输出。应用同时提供自动模式和引导模式，默认使用引导模式，并只允许用户在审议开始前切换；引导模式在独立陈述、质疑补充及报告草稿完成后设置阶段检查点，用户可以继续、提交对后续所有参与者可见的指导意见，或取消审议。相比由模型主持人动态选择发言者，该方案具有确定的参与范围、成本上限和终止条件，不依赖额外的模型 API，并使审议过程更容易复现；代价是可能产生重复发言，且暂不根据内容动态跳过不相关的参与者。
