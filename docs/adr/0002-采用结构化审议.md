# 采用结构化审议

MVP 采用结构化审议：所有参与者先独立陈述，再基于完整记录质疑与补充，随后提交修订意见；若满足按需条件则追加一次有严格范围的争议收敛轮，然后由报告 Agent 草拟报告、其他参与者审阅，最后修订一次并形成共同成果。参与者阶段均在阶段内并行执行，并以屏障等待全部调用成功、失败或超时后再进入下一阶段；同一阶段的参与者不会看到彼此尚未完成的输出。应用同时提供自动模式和引导模式，默认使用引导模式且只允许在开始前切换；引导模式在独立陈述、质疑补充、争议判定及报告草稿后设置检查点。相比动态群聊，该方案具有明确的参与范围、成本上限、终止条件和恢复边界；代价是流程更长，且争议判定需要结构化信号和额外整理。
