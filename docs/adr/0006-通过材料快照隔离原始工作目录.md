# 通过材料快照隔离原始工作目录

工作目录是可选输入；未提供时进行纯文本审议。提供工作目录时默认创建临时材料快照并让所有参与者读取同一副本，用户也可以显式选择直接工作目录模式，此时参与者读取原始目录且界面必须警告安全性弱于快照，并在审议档案中记录该选择。Git 快照默认只复制已跟踪文件及用户明确允许的未跟踪文件，非 Git 目录默认排除隐藏内容；版本库元数据、环境变量文件、密钥、证书、认证目录、依赖目录、构建产物及其他内置敏感模式始终排除且不能关闭，用户可以继续追加排除规则。创建前向用户展示包含与排除统计，疑似秘密不得进入界面、正式记录或原始诊断。无论是否使用快照，各 CLI 适配器都应启用对应运行时的只读、计划或禁用写工具选项。审议成功完成后自动清理材料快照；取消或异常退出后保留可恢复快照，直到恢复成功或用户显式处理。孤立临时文件可通过独立入口手动清理，但不得删除审议档案，也不得清理仍被活动或可恢复审议使用的快照。该方案在便利性与隔离强度之间提供显式取舍，同时保持快照为安全默认值。
