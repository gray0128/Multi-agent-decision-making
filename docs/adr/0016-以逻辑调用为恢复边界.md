# 以逻辑调用为恢复边界

TypeScript 审议引擎以一次参与者发言、主持规划、摘要或报告步骤等逻辑调用作为最小恢复边界：启动 CLI 前持久化冻结输入、逻辑调用 ID 和尝试信息，成功返回后原子提交规范化输出；结构化并行阶段保留已完成参与者的结果，恢复时只重跑失败或未完成的逻辑调用，自由讨论也逐回合提交。明确可重试的瞬时错误和 schema 输出错误各只自动重试一次，第二次仍失败就暂停为可恢复状态；引擎不自动移除参与者、降低最低人数、替换 CLI 或调用预设，也不让报告流程在缺少已确认输出时静默继续。崩溃可能发生在外部 CLI 已完成但结果尚未提交的窗口，此时恢复允许产生一次新的调用尝试和潜在重复计费，但同一逻辑调用只能向权威审议记录提交一次；档案保存各尝试的脱敏诊断关系。恢复始终使用原审议方案、预算和冻结输入，不重新组局、不切换模型调用预设，也不依赖 CLI 私有会话。检查点“暂停审议”和第一次 `Ctrl-C` 都保存可恢复状态并终止当前 CLI 子进程；“取消审议”是保留档案但不生成共同成果的不可恢复终态；自由讨论的“结束讨论”只停止新增发言并进入报告流水线。
