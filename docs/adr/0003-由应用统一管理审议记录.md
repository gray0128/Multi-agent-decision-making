# 由应用统一管理审议记录

应用是审议记录的唯一权威来源；每次发言和主持规划都通过对应 CLI 适配器启动新的非交互进程，并显式传入原问题、当前要求和所需历史，而不使用各 CLI 运行时的 `resume` 或 `continue` 私有会话。每个模型调用预设声明上下文预算，Controller 根据下一次实际输入估算是否超限；任一后续调用预算不足时，由报告 Agent 生成一份保留原问题、用户指导、关键论点、证据来源、未决争议及已作决定的统一滚动摘要，之后所有参与者、主持 Agent 和报告 Agent 都使用同一摘要加摘要后的最近发言，不按模型生成互相不一致的版本。摘要生成属于计入总预算且可独立恢复的逻辑调用，完整原始输出始终保留在审议档案中。该选择确保异构参与者基于同一可见上下文工作，使审议可持久化、检查和重放，但会增加调用与输入成本，且摘要可能损失部分细节。
