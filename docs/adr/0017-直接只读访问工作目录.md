# 直接只读访问工作目录

项目审议不再创建材料快照：只有用户显式传入 `--workspace <path>` 时，组局器和所有参与者才从组局阶段开始共同直接只读访问该目录；命令不隐式使用当前目录，也不再提供 `--direct-workspace` 或额外确认，显式路径本身就是读取授权。应用在标准错误提示直接读取风险，并把规范化绝对路径与读取模式写入审议档案。各 CLI 适配器必须启用并验证对应运行时的只读、计划或禁用写工具模式，不满足最低只读能力的适配器或调用预设不得用于项目审议。该选择避免复制大型目录的时间、空间和清理成本，也删除快照恢复与孤立临时文件管理；代价是应用不再通过快照排除秘密、依赖目录或构建产物，“只读”只限制修改而不限制 CLI 能看到的文件，用户必须把显式指定目录视为对所选 CLI 的读取授权。
