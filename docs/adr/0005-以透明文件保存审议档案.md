# 以透明文件保存审议档案

每次审议保存为独立目录，不引入数据库。TypeScript 版本使用带 `schema_version` 的 `manifest.json` 和原子替换的 `state.json` 保存身份与可恢复状态，使用只追加的 `events.jsonl`、`transcript.jsonl` 和 `diagnostics.jsonl` 保存实时事件、权威发言与脱敏诊断，并以 `report.md` 保存最终共同成果；该格式不兼容 Python 版本的历史档案。透明文件便于用户直接检查、备份、版本管理和删除，也让本地观察服务可以通过文件变化提供实时展示；代价是跨审议查询和并发写入能力有限，只有在历史检索或并发需求成立后才考虑 SQLite。
