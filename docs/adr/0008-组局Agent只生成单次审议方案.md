# 组局 Agent 只生成单次审议方案

应用提供默认关闭、按次启用的组局功能。用户选择启用后，可以在审议开始前指定一个已注册的 Agent 配置作为组局 Agent，由其根据问题和 Agent 注册表提出参与者、临时角色及报告 Agent 等单次审议方案，并在用户确认或修改后执行；手动配置和组局建议都必须经过最终确认才能启动。组局 Agent 只能引用注册表中已有的 Agent 配置，不能改写全局 `agents.toml`，也不能生成新的可执行命令、模型参数或秘密。该边界在保留自动组队便利性的同时，避免模型输出扩大进程执行权限或永久改变可信配置。
