# MVP 使用原生 DevUI

MVP 直接使用 Microsoft Agent Framework DevUI 作为本地交互、流式展示和调试追踪界面，不开发独立前端。工作流以结构化输入 schema 声明问题、参与者、报告 Agent、工作目录、审议模式、争议收敛策略和可选组局 Agent，由 DevUI 自动生成启动表单；启用组局功能时，组局建议仍须经过用户确认。审议运行中的四个阶段检查点使用工作流暂停询问。DevUI 固定监听 `127.0.0.1` 并保留启动时生成的随机 Bearer Token 认证，不开放局域网或公网监听。该方案以较低成本提供必要的配置与观察能力，并避免在流程尚未稳定前维护自定义 UI，但只定位于本地开发工具而非生产级服务。
