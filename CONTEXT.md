# 多 Agent 审议

本领域描述一个本地系统：一个或多个独立安装的 AI 编程 CLI 通过多个可配置参与身份围绕审议议题进行审议，并形成共同成果。

## 领域语言

**CLI 运行时**：
一个已独立安装并完成认证、能够以非交互方式调用的 AI 命令行程序。
_避免使用_：Agent、模型、模型 API

**CLI 适配器**：
将应用的统一调用请求转换为某类 CLI 运行时参数、输入和输出事件，并验证模型调用预设是否受该 CLI 支持的集成组件。
_避免使用_：Agent、命令模板、模型供应商

**CLI 配置**：
一个长期可信配置，声明 CLI 适配器、可执行程序、允许参数、能力限制和可选择的模型调用预设，不包含审议角色。
_避免使用_：Agent 配置、审议 Agent、角色模板

**模型调用预设**：
CLI 配置中明确允许选择、并由对应 CLI 适配器验证的模型及推理设置组合；同一模型可以拥有不同思考等级的多个预设。
_避免使用_：裸模型名、动态模型、任意 CLI 参数

**CLI 注册表**：
由用户维护、应用使用的可信 CLI 配置集合，是可执行程序、允许参数和模型选择的权限边界。
_避免使用_：Agent 注册表、参与者列表、审议方案

**审议 Agent**：
在一次审议的组局阶段生成、只在该次审议中存在的参与身份，由唯一 ID、一个 CLI 配置、一个模型调用预设和角色描述组成。
_避免使用_：Agent 配置、CLI 运行时、长期角色

**审议议题**：
一次审议需要回答、决定或评审的事项，可以是问题、待决策事项或待评审方案，不要求采用问句形式。
_避免使用_：问题、Prompt、任务描述

**审议**：
多个审议 Agent 围绕一个审议议题开展的、有明确边界的多轮交流，目标是共同产出成果。
_避免使用_：聊天、工作流运行、任务

**活动审议**：
已经取得当前应用数据根目录的运行权、尚未完成或退出，并且可以推进及写入状态的一次审议；同一应用数据根目录最多存在一个活动审议。
_避免使用_：历史档案、可恢复审议、页面会话

**只读审议**：
参与者可以分析提供给它的上下文并生成文本，但不能修改用户文件或执行会改变外部状态的操作。
_避免使用_：沙箱模式、无工具模式

**结构化审议**：
所有参与者按预先规定的阶段参与同一次审议，每个阶段都有明确目的和结束条件。
_避免使用_：自由聊天、抢话、动态主持

**自由讨论**：
参与者共享当前审议记录，但不遵循预先规定的发言阶段；每个发言回合只有一名参与者发言，发言顺序和结束时机动态确定，同时受明确的资源上限约束。
_避免使用_：结构化审议、无限对话、无约束聊天

**发言回合**：
自由讨论中由一名参与者基于当前权威审议记录完成的一次发言，是动态调度和资源计数的最小单位。
_避免使用_：阶段、并发发言、完整一轮

**覆盖周期**：
自由讨论开始时保证每位参与者恰好获得一个发言回合的初始周期；主持 Agent 只决定尚未发言参与者之间的顺序。
_避免使用_：固定审议阶段、开放讨论、检查窗口

**检查窗口**：
覆盖周期结束后，以参与者数量为长度划分的一组连续发言回合；窗口结束时检查收敛并允许用户决定后续动作，不要求窗口覆盖所有参与者。
_避免使用_：阶段检查点、覆盖周期、完整一轮

**主持 Agent**：
自由讨论中根据当前审议记录规划后续检查窗口并判断是否建议继续的审议 Agent，必须是参与者；其调度判断不属于发言回合，也不作为参与者观点。
_避免使用_：组局 Agent、报告 Agent、用户

**共同成果**：
审议完成后形成的最终结论或报告，是整个审议的交付物，而不是任一参与者的原始发言。
_避免使用_：最终回复、汇总消息

**审议记录**：
由应用统一维护的权威交流历史，包含审议议题、阶段或回合以及各参与者的可见发言。
_避免使用_：CLI 会话、聊天记录、Agent 记忆

**审议摘要**：
当完整审议记录超过参与者的上下文预算时，由报告 Agent 生成、供所有后续参与者共同使用的压缩上下文。
_避免使用_：裁剪记录、摘要记忆、报告草稿

**参与者**：
在一次审议开始前由用户确认加入方案的审议 Agent；一次审议至少需要两名参与者。
_避免使用_：模型列表、CLI 列表、成员

**审议方案**：
只属于一次审议的已确认设置，包含组局器、参与者、报告 Agent、主持 Agent、审议模式和工作目录。
_避免使用_：CLI 注册表、长期配置、工作流配置

**方案确认**：
组局阶段生成或经用户修改、通过校验与预检的审议方案获得执行授权；修改或重新组局后必须再次确认，自动化调用也必须显式授权，且授权方式必须进入审议档案。
_避免使用_：跳过组局、隐式同意、安全校验

**组局阶段**：
每次审议在正式交流前必须执行的固定步骤，由组局器根据审议议题和 CLI 注册表生成审议 Agent 及审议方案，再交由用户确认。
_避免使用_：可选组局、审议阶段、自由讨论

**组局器**：
组局阶段使用的 CLI 配置与模型调用预设组合；应用提供默认组局器，用户可以为单次审议指定另一项允许组合。
_避免使用_：组局 Agent、主持 Agent、报告 Agent

**可用模型调用预设**：
其 CLI 运行时、模型及推理设置和非交互调用已经通过应用健康检查的模型调用预设。
_避免使用_：已安装 Agent、在线 Agent、健康 CLI

**报告 Agent**：
一次审议中负责起草共同成果，并根据其他参与者的审阅意见完成最终修订的审议 Agent。
_避免使用_：主持人、裁判、最终发言者

**报告审阅**：
报告草稿形成后，除报告 Agent 外的参与者针对事实错误、重要遗漏和证据不足开展的一次有边界检查。
_避免使用_：新一轮讨论、投票、共同编辑

**关键未决争议**：
修订意见形成后仍未解决，并且不同答案会实质改变共同成果结论、建议或风险判断的分歧。
_避免使用_：一般异议、观点差异、待办问题

**争议收敛轮**：
在修订意见之后按需触发、只处理已识别关键未决争议的一次附加审议阶段。
_避免使用_：第四轮讨论、自由辩论、重复修订

**收敛结果**：
关键未决争议经过收敛后形成的状态，分为已解决、条件性一致或仍未解决，不以多数投票或强制共识决定。
_避免使用_：投票结果、胜方观点、统一意见

**工作目录**：
用户为一次审议显式指定、供组局器和参与者共同直接只读访问的本地目录；未提供时进行纯文本审议。
_避免使用_：项目、仓库、沙箱

**命令行入口**：
供用户或其他 Coding Agent 发起、控制、恢复和获取审议结果的操作入口，与审议控制台共享相同的审议规则和档案。
_避免使用_：CLI 运行时、CLI 适配器、无界面模式

**审议控制台**：
由本地服务提供的网页操作入口，可以发起独立的审议进程，并展示审议档案、实时状态和当前检查点；审议进程仍是运行状态和正式档案的唯一所有者。
_避免使用_：审议观察页、DevUI、管理后台

**检查点响应**：
用户针对当前等待中的检查点提交的一次性决定，可以继续、补充指导、结束自由讨论、暂停审议、取消审议或选择允许的收敛动作；过期或重复响应不属于有效审议输入。
_避免使用_：通用页面操作、审议配置、聊天消息

**结束讨论**：
停止自由讨论中的新增发言并进入共同成果流水线，不结束整个审议。
_避免使用_：取消审议、暂停审议、强制退出

**暂停审议**：
保存当前可恢复状态并退出活动审议进程，之后可以从同一审议方案和冻结输入继续。
_避免使用_：取消审议、失败、结束讨论

**取消审议**：
将审议置为不可恢复的终态，不生成共同成果但保留已有档案供审计。
_避免使用_：暂停审议、结束讨论、删除档案

**审议档案**：
一次审议自动落盘形成的独立资料集合，包含状态、完整审议记录、最终报告以及调用诊断信息。
_避免使用_：数据库记录、日志目录、聊天导出

**阶段检查点**：
引导模式下位于两个审议阶段之间的暂停点，用户可以继续、补充指导意见或取消审议。
_避免使用_：审批、插话、断点

**指导意见**：
用户在阶段检查点提交并写入审议记录的补充内容，从下一阶段起对所有参与者可见。
_避免使用_：用户消息、临时提示词、主持意见
