# 多 Agent 审议

本领域描述一个本地系统：多个独立安装的 AI 编程 CLI 围绕用户问题进行审议，并形成共同成果。

## 领域语言

**CLI 运行时**：
一个已独立安装并完成认证、能够以非交互方式调用的 AI 命令行程序。
_避免使用_：Agent、模型、模型 API

**CLI 适配器**：
将应用的统一调用请求转换为某类 CLI 运行时参数、输入和输出事件的集成组件。
_避免使用_：Agent、命令模板、模型供应商

**Agent 配置**：
一个可命名和选择的独立身份，由 CLI 适配器、模型或服务商、可执行路径、参数及角色指令共同定义；同一 CLI 适配器可以拥有多个 Agent 配置。
_避免使用_：CLI Agent、模型、适配器实例

**Agent 注册表**：
由用户维护、应用启动时加载的可信 Agent 配置集合，是审议可选身份和允许参数的边界。
_避免使用_：审议配置、参与者列表、动态 Agent

**审议**：
多个 Agent 配置围绕一个问题开展的、有明确边界的多轮交流，目标是共同产出成果。
_避免使用_：聊天、工作流运行、任务

**只读审议**：
参与者可以分析提供给它的上下文并生成文本，但不能修改用户文件或执行会改变外部状态的操作。
_避免使用_：沙箱模式、无工具模式

**结构化审议**：
所有参与者按预先规定的阶段参与同一次审议，每个阶段都有明确目的和结束条件。
_避免使用_：自由聊天、抢话、动态主持

**共同成果**：
结构化审议完成后形成的最终结论或报告，是整个审议的交付物，而不是任一参与者的原始发言。
_避免使用_：最终回复、汇总消息

**审议记录**：
由应用统一维护的权威交流历史，包含问题、阶段以及各参与者的可见发言。
_避免使用_：CLI 会话、聊天记录、Agent 记忆

**审议摘要**：
当完整审议记录超过参与者的上下文预算时，由报告 Agent 生成、供所有后续参与者共同使用的压缩上下文。
_避免使用_：裁剪记录、摘要记忆、报告草稿

**参与者**：
在一次审议开始前由用户选中的可用 Agent 配置；一次审议至少需要两名参与者。
_避免使用_：模型列表、CLI 列表、成员

**审议方案**：
只属于一次审议的已确认设置，包含参与者、临时角色、报告 Agent、审议模式和工作目录。
_避免使用_：Agent 注册表、agents.toml、工作流配置

**组局 Agent**：
用户可选的一项 Agent 配置，用于根据问题和 Agent 注册表提出审议方案，但无权修改注册表或生成新的命令及模型参数。
_避免使用_：主持人、报告 Agent、配置生成器

**可用 Agent 配置**：
其 CLI 运行时、模型或服务商参数及非交互调用已经通过应用健康检查的 Agent 配置。
_避免使用_：已安装 Agent、在线 Agent、健康 CLI

**报告 Agent**：
一次审议中负责起草共同成果，并根据其他参与者的审阅意见完成唯一一次修订的参与者。
_避免使用_：主持人、裁判、最终发言者

**报告审阅**：
报告草稿形成后，除报告 Agent 外的参与者针对事实错误、重要遗漏和证据不足开展的一次有边界检查。
_避免使用_：新一轮讨论、投票、共同编辑

**关键未决争议**：
修订意见形成后仍未解决，并且不同答案会实质改变共同成果结论、建议或风险判断的分歧。
_避免使用_：一般异议、观点差异、待办问题

**争议收敛轮**：
在修订意见之后按需触发、只处理已识别关键未决争议的一次附加审议阶段。
_避免使用_：第四轮讨论、自由辩论、重复修订

**收敛结果**：
关键未决争议经过收敛后形成的状态，分为已解决、条件性一致或仍未解决，不以多数投票或强制共识决定。
_避免使用_：投票结果、胜方观点、统一意见

**工作目录**：
用户可为一次审议选择的本地目录；未提供时进行纯文本审议，提供后默认从中制作材料快照，也可由用户显式允许参与者直接只读访问。
_避免使用_：项目、仓库、沙箱

**材料快照**：
应用根据工作目录制作、供所有参与者共同读取的临时副本，是提供工作目录时的默认隔离方式。
_避免使用_：工作副本、临时仓库、缓存目录

**直接工作目录**：
用户显式选择后由参与者直接只读访问的原始工作目录，不经过材料快照隔离。
_避免使用_：无沙箱模式、写入模式、当前目录

**命令行入口**：
供用户或其他 Coding Agent 从终端发起、观察和获取审议结果的应用接口，与 DevUI 共用同一审议引擎。
_避免使用_：CLI 运行时、CLI 适配器、无界面模式

**孤立临时文件**：
因应用异常退出等原因遗留、且不再被活动或可恢复审议使用的材料快照及其他临时数据。
_避免使用_：审议档案、垃圾文件、缓存

**审议档案**：
一次审议自动落盘形成的独立资料集合，包含状态、完整审议记录、最终报告以及调用诊断信息。
_避免使用_：数据库记录、日志目录、聊天导出

**阶段检查点**：
引导模式下位于两个审议阶段之间的暂停点，用户可以继续、补充指导意见或取消审议。
_避免使用_：审批、插话、断点

**指导意见**：
用户在阶段检查点提交并写入审议记录的补充内容，从下一阶段起对所有参与者可见。
_避免使用_：用户消息、临时提示词、主持意见
